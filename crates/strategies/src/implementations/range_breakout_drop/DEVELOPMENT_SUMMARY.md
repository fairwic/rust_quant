## 策略设计概要

### 入场逻辑（5层过滤）

```
第1层：震荡识别
└─> 波动率在 0.5% - 3.0% 范围内

第2层：突破确认
└─> 价格突破下边界 + 实体≥55% + 移动≥0.8ATR + 放量≥1.5倍

第3层：趋势过滤
└─> 价格 < EMA50 + RSI ≥ 40

第4层：K线方向
└─> 必须是阴线

第5层：风控
└─> 交易所白名单 + 冷却期
```

### 止损止盈

```
入场价: P
震荡上边界: H
风险距离: R = H + 1.5ATR - P

止损: H + 1.5ATR
止盈T1: P - 1.0R
止盈T2: P - 2.0R  
止盈T3: P - 3.5R
```

## 遇到的问题与解决

### 问题1: 字段名不匹配
**症状**: `CandleItem.volume` 字段不存在  
**原因**: 实际字段名是 `v`  
**解决**: 全局替换 `c.volume` → `c.v`

### 问题2: SignalResult 结构不匹配
**症状**: `price`, `timestamp`, `reason` 字段不存在  
**原因**: 实际使用 `open_price`, `ts`, `filter_reasons`  
**解决**: 参考 bear_short_stack 策略，使用正确的字段名

### 问题3: StrategyType::Custom 需要参数
**症状**: `StrategyType::Custom` 编译错误  
**原因**: Custom 是元组变体，需要一个 u32 参数  
**解决**: 使用 `StrategyType::Custom(1001)`

### 问题4: 回测未产生信号
**症状**: `trade_records` 和 `filtered_signals` 都为空  
**原因**: 测试数据波动率太小（0.20% < 0.5%），不满足震荡识别条件  
**解决**: 通过单元测试验证逻辑正确，需要使用真实数据

## 性能考虑

- **指标计算**: EMA, ATR, RSI 都是 O(n) 复杂度

- **内存占用**: 快照结构约 200 bytes
- **冷却机制**: 避免连续开仓，降低交易频率
