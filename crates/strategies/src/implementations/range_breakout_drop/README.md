# 震荡结束突破下跌策略 (Range Breakout Drop Strategy)

## 策略概述

**策略类型**: 做空策略  
**版本**: v1  
**策略KEY**: `range_breakout_drop_v1`  
**状态**: 实验阶段 - 代码实现完成，待回测验证

## 策略逻辑

### 核心思路
识别震荡区间结束后向下突破的做空机会。震荡区间为策略提供明确的止损位（区间上边界），突破下边界后有充足的下跌空间。

### 入场条件（必须全部满足）

1. **震荡识别**
   - 过去N根K线（默认20根）价格在一定范围内横盘
   - 震荡幅度在 0.5% - 3.0% 之间（可配置）
   - 波动不能太大（超过3%不认为是震荡）
   - 波动不能太小（低于0.5%流动性不足）

2. **突破确认**
   - 当前K线收盘价突破震荡区间下边界
   - 突破K线实体比例 ≥ 0.55（过滤十字星）
   - 突破移动距离 ≥ 0.8 ATR
   - 突破成交量 ≥ 震荡期平均成交量的1.5倍

3. **趋势过滤**（可选）
   - 价格低于慢速EMA（默认50周期）
   - RSI ≥ 40（避免在超卖区做空）

4. **K线方向**
   - 必须是阴线（收盘价 < 开盘价）

### 止损与止盈

- **止损**: 震荡区间上边界 + 1.5 ATR
- **止盈**: 
  - T1: 1R（风险距离的1倍）
  - T2: 2R
  - T3: 3.5R

### 风控机制

- 冷却期: 6根K线（避免连续开仓）
- 交易所白名单: Binance, OKX, Hyperliquid
- 只做空，不做多

## 设计决策记录

### 为什么选择震荡突破下跌？
- ✅ 震荡区间提供明确的止损位
- ✅ 适合当前熊市环境
- ✅ 与现有策略（bear_short_stack）形成互补
- ✅ 可以复用大量现有基础设施

### 为什么只做空？
- 当前产品定位是熊市策略栈
- 降低策略复杂度，专注做空逻辑
- 震荡突破上涨需要完全不同的确认逻辑

### 为什么使用ATR而不是固定点数？
- 适应不同波动率环境
- 与其他策略保持一致
- 便于跨交易对应用
