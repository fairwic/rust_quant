## 市场一致性

趋势策略四市场平均 R：BTC -0.064、ETH -0.028、SOL -0.099、BCH -0.193。

区间策略四市场平均 R：BTC -0.346、ETH -0.563、SOL -0.427、BCH -0.256。

两个策略在四市场基础成本后均为负。趋势在 ETH 上最接近盈亏平衡，但仍为负，且两倍成本后恶化到 -0.131R，不能据此创建币种专用 Challenger。

## 模型竞赛

- 趋势和区间的 one-standard-error 选择均回到 M0 无过滤基线。
- 趋势 M1 固定规则验证均值从 M0 的 -0.114R 改善到 -0.080R，但仍为负，且与 M0 位于同一标准误范围内，不能构成优势。
- 逻辑回归、CART 和 Forest 均未保留验证交易，因覆盖率为 0 被淘汰。
- 没有生成可晋级的 RuntimeManifest Challenger。

当前逻辑回归以 `net_r > 0` 为二分类标签并固定使用 `0.5` 概率阈值。该阈值没有直接优化成本后期望 R，是下一轮训练协议需要修正的模型目标缺口；本报告不在查看结果后临时改阈值重写本批次结论。

## 成本/R 诊断

- 趋势 `cost_r <= 0.10`：500 笔，基础平均 -0.054R、PF 0.915；两倍成本 -0.113R，仍失败。
- 区间 `cost_r <= 0.10`：仅 3 笔，基础平均 -0.166R；没有形成可研究的低成本正期望子群。
- 1h 相比 15m 降低了趋势策略的成本拖累，但没有把候选语义转化为正期望。

## 决策

1. `pa_trend_1h` 与 `pa_range_1h` 不进入 Shadow/Paper。
2. 1h 与 15m 均未证明当前 v1 候选结构具备统计优势；不得打开密封 OOS 为失败训练假设继续调参。
3. 下一轮先修正训练目标：模型 keep 判定应直接估计或优化成本后期望 R，并在训练折内选择阈值，验证折只评分。
4. 新结构假设应优先解释趋势入场质量，而不是继续收紧成本门槛；四市场低成本子群仍为负。
5. 接入历史资金费率并冻结新一轮协议前，不生成完整 PromotionRecord，也不注册到默认生产执行器。
